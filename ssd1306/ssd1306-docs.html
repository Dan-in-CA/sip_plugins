About the SSD1306 Plugin

This plugin has been created to provide a simple output to display current state and to tie to the keypad plugin for entry display (keypad plugin not required to use this plugin).

Hardware Requirements

Only the 128x64 SSD1306 variant is supported by this plugin.

Installing the Hardware

1.       Look at the back of your SSD1306 and note what address the board is set to. There are normally two options on the board for either 78 or 7A. The pads soldered together or jumped using a resistor selects the hardware address for the display. Normally 78 is selected, and this is the default address set for this plugin. You may change this value at any time through the SSD1306 Plugin Settings page on the SIP web interface.

2.       Unplug your raspberry pi, and then wire your SSD1306 to pins for I2C.1. For most raspberry pi hardware, this is at pins 3 and 5 of the header. Refer to the schematics for your particular raspberry pi version before selecting the pins to wire to.

3.       Plug in your raspberry pi and install the I2C tools by entering the command sudo apt-get install i2c-tools

4.       Execute the command i2cdetect -y 1

a.       If your hardware is setup for address 78, you should see 3C in the table. If your hardware is setup for address 7A, you should see 3D in the table. This confirms that your display is wired properly.

5.       Make sure the python smbus package is installed by running sudo apt-get install python-smbus

You should now be ready to install the SSD1306 plugin.

Note: it is not recommended to increase the I2C BAUD rate since the I2C bus is shared by the hardware controlling sprinkler hardware. Parts of your hardware may only be rated to 100kHz which is the default speed selected for the Raspberry Pi I2C busaspberry Pi. For example, the max rated BAUD rate for the PCF8591T A/D and D/A controller is 100kHz.

SSD1306 Settings

The settings page for the SSD1306 has the following fields.

       Idle Display Timeout in Seconds (0 for always on)

o   The system is considered idle when it is not currently running any programs or stations. This is the number of seconds to wait once in idle mode before shutting off the screen. This helps to limit burn-in over time. A value of 0 here will disable this timeout.

o   When the screen is idled, it will turn back on due to any change to the system state. This includes things like rain delay deactivating or changing the water level.

       Hardware Address in Hex

o   This is the address printed on the hardware. The retrieved value in i2cdetect will be this value shifted 1 bit to the right ex: a value of 78 (0b01111000) here will correspond to 3C (0b00111100) in i2cdetect.

       I2C Bus

o   This is the I2C bus the display is connected to. I decided to make this non-modifiable in order to limit confusion. I see no current use-case where this should be anything other than 1.

       Width in Pixels and Height in Pixels

o   This is the displays width and height in number of pixels. These values are also non-modifiable since a lot of the internals assume the size of the screen. I dont have any other configuration of this hardware, so I didnt feel comfortable making this more dynamic since I have no way to properly test it. If you want to use a different screen size, feel free to fork the github project and make code changes as needed.

SSD1306 Signaling

The SSD1306 plugin may be controlled by external plugins using the ssd1306_wake and ssd1306_display signals. You can enable signals in your plugin by importing signal from SIPs blinker module. Refer to the signaling_examples.py file in the plugins directory of SIP for examples of how this works.

ssd1306_wake Signal

This signal takes no arguments and simply tells the SSD1306 plugin to wakeup and reset its idle display timer.

Code example:

ssd1306_wake_signal = signal(u"ssd1306_wake")
ssd1306_wake_signal.send()

ssd1306_sleep Signal

This signal takes no arguments and simply tells the SSD1306 plugin to go to sleep.

Code example:

ssd1306_sleep_signal = signal(u"ssd1306_sleep")
ssd1306_sleep_signal.send()

ssd1306_display Signal

This signal enables you to take over the display from another plugin and show whatever text you would like into some specified drawing area. All other pixels in the selected area will be cleared. It takes only the following keyed arguments. All other keyed arguments will be ignored.

       activator

o   The plugin name that activated this signal.

       screen_id (optional, default=default)

o   The screen ID to write to. Setting this value is useful if you would like to build your own menu system and uniquely identify displays.

       cancel (optional, default=False)

o   Set to True to cancel the previously displayed screen for the given activator and screen_id. All other arguments are ignored if this is True. This pops the display from the display stack if it is found on the stack. Nothing happens if the screen is not found on the stack.

       wake (optional, default=True)

o   Set to True in order to automatically wake the display whenever this screen is shown.

       txt (optional, default=u)

o   The text to display.

       row_start (optional, default=0)

o   The 0-based 8-bit row where the text will start to display. Text will write downward from here. Value must be an integer in the range [0,7].

       row_end (optional, default=None)

o   The 0-based 8-bit row where the text is limited to on the display (inclusive). Value must be an integer in the range [0,7] or None. If this is set to None, the internals will compute a value based on the number of \n found in txt and max_text_size.

       col_start (optional, default=0)

o   The 0-based column where text will start to display. Value must be an integer in the range [0,127].

       col_end (optional, default=127)

o   The 0-based column where the text is limited to on the display (inclusive). Value must be an integer in the range [0,127].

       min_text_size (optional, default=1)

o   Minimum text size integer. This may be an integer in the range [1,8].

       max_text_size (optional, default=1)

o   Minimum text size integer. This may be an integer in the range [1,8].

       text_size (optional, default=None)

o   If specified, min and max text size is set to this value. This may be an integer in the range [1,8].

       justification (optional, default=LEFT)

o   Horizonal justification option for the text. This may be one of LEFT, CENTER, or RIGHT.

       append (optional, default=False)

o   Set to True to append this text to the previously defined activator and screen_id. If this is False, the previously defined screen will be cleared before writing to it.

       delay (optional, default=1)

o   The number of seconds to display this screen until automatically popping it from the stack. This may be set to None in order to never pop this screen until subsequently cancelled using the cancel key. The delay for the screen will always be set to the last set value when this signal is used multiple times.

Code example:

ssd1306_wake_signal = signal(u"ssd1306_display")
# Clear screen and show Hello, world!
ssd1306_wake_signal.send(
activator=umy_plugin,
screen_id=uscreen_1,
txt=uHello, world!,
row_start=0,
text_size=2,
justification=uCENTER,
append=False,
delay=None)
# Append fee fi foo fum to this screen
ssd1306_wake_signal.send(
activator=umy_plugin,
screen_id=uscreen_1,
txt=ufee fi foo fum,
row_start=5,
text_size=2,
justification=uLEFT,
append=False,
delay=None)
# Pop the screen from the display stack
ssd1306_wake_signal.send(
activator=umy_plugin,
screen_id=uscreen_1,
cancel=True)
